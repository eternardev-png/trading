Структура проекта "AlgoResearch Lab"

Проект представляет собой комплексное решение для алгоритмических исследований и визуализации финансовых данных, состоящее из серверной части (Python) и клиентского интерфейса (React/Vite).

## 1. Корневая директория (Root)
Содержит скрипты управления проектом и конфигурационные файлы:
- `start.ps1`, `setup_env.ps1`: PowerShell скрипты для настройки окружения и запуска приложения.
- `full_sync.bat`, `sync_github.bat`: Пакетные файлы для синхронизации с репозиторием.
- `COMMANDS.txt`: Полезные команды для работы с проектом.
- `verify_result.txt`, `debug_output.txt`: Логи и результаты проверок.

## 2. Серверная часть (Backend) - папка `backend/`
Отвечает за бизнес-логику, обработку данных и расчет индикаторов.
Основные компоненты:
- `server.py`: Основной файл запуска сервера (вероятно, предоставляет API для фронтенда).
- `data_loader.py`: Модуль загрузки исторических и рыночных данных.
- `indicators.py` (и папка `indicator/`): Реализация алгоритмов и технических индикаторов.
- `synthetic_engine.py`: Движок для генерации синтетических данных для тестирования.
- `backtester.py`: Модуль для тестирования стратегий на исторических данных.
- `tv_loader.py`: Загрузчик данных (возможно, связанных с TradingView).

Утилиты отладки и проверки:
- Множество скриптов `verify_*.py` (например, `verify_indicators.py`, `verify_m2_loader.py`) для тестирования конкретных модулей.
- Скрипты `debug_*.py` для отладки компонентов.
- `check_deps.py`: Проверка зависимостей.

## 3. Клиентская часть (Frontend) - папка `frontend/`
Современный веб-интерфейс для взаимодействия с пользователем.
Технологический стек: React, Vite.
Ключевые файлы:
- `vite.config.js`: Конфигурация сборщика Vite.
- `package.json`: Управление JavaScript-зависимостями.

Исходный код (`frontend/src/`):
- `App.jsx`, `main.jsx`: Точка входа в приложение и основной компонент.
- `components/`: Библиотека UI-компонентов (графики, панели управления, виджеты).
- `services/`: Модули для общения с сервером (API-клиенты).
- `stores/`: Управление состоянием приложения (хранилище данных).
- `styles/`: Глобальные стили стилизации (SCSS/CSS).
- `utils/`: Вспомогательные функции и утилиты.

## 4. Дополнительные директории
- `data/`: Хранение локальных данных (используется и в корне, и в backend).
- `Example UI`, `frontend-example`: Примеры реализации интерфейсов или старые версии.
- `java_ui/`: Вероятно, альтернативный или устаревший интерфейс на Java.
